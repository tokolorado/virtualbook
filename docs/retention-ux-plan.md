# Retention UX

Stan: 2026-05-07.

Ten dokument opisuje bezpieczne pomysły retencyjne. Nie mieszamy gamifikacji z krytyczną logiką kursów, kuponów, ledgera i settlementu.

## Małe Bezpieczne Poprawki

- Daily streak przy profilu lub saldzie VB.
- Delikatny komunikat powrotu po dłuższej przerwie.
- Mikrocopy przy kuponie, bez presji finansowej.
- Poczucie progresu: poziom konta, sezon albo pasek aktywności.
- Mocniejsze wizualne pokazanie VB jako waluty gry.
- CTA do powrotu, gdy są nowe mecze, aktywne kupony albo aktywność znajomych.

## Osobny Etap Gamifikacji

Te elementy wymagają osobnych tabel, audytu i reguł:

- quizy,
- misje,
- sklep VB,
- dzienne koło fortuny,
- jawne prawdopodobieństwa nagród,
- jackpot 5000 VB z prawdopodobieństwem 1 na 10000 zakręceń.

## Zasady Wdrożenia

- Każda nagroda VB musi przechodzić przez ledger.
- Każda losowość musi mieć audytowalny zapis wyniku i źródła losowania.
- UI ma wzmacniać powrót i progres, ale nie udawać realnych pieniędzy.
- Retencja nie może wpływać na pricing, settlement ani idempotencję kuponów.
